# 個人情報保護と年度管理ガイド

## 概要

このガイドでは、卒業生の個人情報保護を考慮しながら、年度ごとに学生データを追加・管理する方法について説明します。

## 現在の実装状況

### 個人情報の扱い
- **発表会終了後**: 氏名・英字表記・学籍番号を非表示
- **データ構造**: `students.json`から`name`と`nameEn`フィールドを削除済み
- **表示制御**: HTMLとJavaScriptでコメントアウト済み

## 推奨される対応方法

### 方法1: 年度別データファイル + 設定ファイル（推奨）

**メリット**:
- 年度ごとにデータを分離できる
- 過去年度のデータを簡単に非公開化できる
- 設定ファイルで表示制御を一元管理できる

**実装例**:

```
data/
├── students.json          # 現在年度のデータ（発表会前）
├── students_2025.json    # 2025年度のデータ（発表会後は非公開）
├── students_2026.json    # 2026年度のデータ
└── config.json           # 表示設定ファイル
```

**config.json**:
```json
{
  "privacySettings": {
    "2025": {
      "hidePersonalInfo": true,
      "hideAfterDate": "2026-01-28"
    },
    "2026": {
      "hidePersonalInfo": false,
      "hideAfterDate": "2027-01-28"
    }
  },
  "currentYear": 2026
}
```

### 方法2: データにフラグを追加（シンプル）

**メリット**:
- 1つのファイルで管理できる
- 実装が簡単

**実装例**:

```json
{
  "id": 1,
  "grade": 2025,
  "studentId": "1522008",
  "title": "研究題目",
  "privacy": {
    "hidePersonalInfo": true,
    "hideAfterDate": "2026-01-28"
  }
}
```

### 方法3: 個人情報を別ファイルに分離（最も安全）

**メリット**:
- 個人情報を完全に分離できる
- 公開リポジトリに個人情報を含めない

**実装例**:

```
data/
├── students_public.json   # 公開用（個人情報なし）
└── students_private.json  # 非公開用（個人情報あり、.gitignoreに追加）
```

## 推奨実装: 方法1（年度別ファイル + 設定ファイル）

### 1. ディレクトリ構造

```
data/
├── students.json          # 現在年度（自動生成）
├── archive/               # 過去年度のアーカイブ
│   ├── students_2025.json
│   └── students_2026.json
└── config.json           # 表示設定
```

### 2. config.jsonの構造

```json
{
  "currentYear": 2026,
  "years": [
    {
      "year": 2025,
      "presentationDate": "2026-01-27",
      "hidePersonalInfoAfter": "2026-01-28",
      "hidePersonalInfo": true,
      "dataFile": "archive/students_2025.json"
    },
    {
      "year": 2026,
      "presentationDate": "2027-01-27",
      "hidePersonalInfoAfter": "2027-01-28",
      "hidePersonalInfo": false,
      "dataFile": "students.json"
    }
  ]
}
```

### 3. JavaScriptの実装

```javascript
// 年度ごとの個人情報表示制御
function shouldHidePersonalInfo(year) {
    const config = getConfig(); // config.jsonを読み込む
    const yearConfig = config.years.find(y => y.year === year);
    
    if (!yearConfig) return true; // 設定がない場合は非表示
    
    // 日付ベースの制御
    if (yearConfig.hidePersonalInfoAfter) {
        const today = new Date();
        const hideDate = new Date(yearConfig.hidePersonalInfoAfter);
        if (today >= hideDate) {
            return true;
        }
    }
    
    // フラグベースの制御
    return yearConfig.hidePersonalInfo || false;
}
```

## 年度ごとの更新手順

### 新年度の追加手順

1. **前年度のデータをアーカイブ**
   ```bash
   mv data/students.json data/archive/students_2025.json
   ```

2. **config.jsonを更新**
   - 前年度の設定を追加（`hidePersonalInfo: true`）
   - 新年度の設定を追加

3. **新年度のデータを作成**
   - `data/students.json`に新年度のデータを追加
   - 個人情報（`name`, `nameEn`）を含める

4. **発表会後**
   - `config.json`で前年度の`hidePersonalInfo`を`true`に設定
   - または、`students.json`から個人情報フィールドを削除

## セキュリティのベストプラクティス

### 1. Git管理の考慮

**推奨**:
- 個人情報を含むファイルは`.gitignore`に追加
- または、個人情報を削除したバージョンのみをコミット

**.gitignore例**:
```
data/students_private.json
data/archive/*_private.json
```

### 2. データの分離

**推奨**:
- 公開用データと非公開用データを分離
- 非公開データはローカル環境でのみ管理

### 3. 自動化スクリプト

**推奨**:
- 年度更新を自動化するスクリプトを作成
- 発表会日を設定すると自動的に非表示になる

## 実装例: 自動化スクリプト

年度管理スクリプト (`scripts/manage_year.py`) が実装済みです。

### 使用方法

```bash
# 設定を確認
python scripts/manage_year.py show-config

# 新年度を追加
python scripts/manage_year.py add-year --year 2026 --presentation-date 2027-01-27

# 発表会後に個人情報を非表示にする
python scripts/manage_year.py hide-info --year 2025

# 年度データをアーカイブに移動
python scripts/manage_year.py archive --year 2025
```

詳細は `scripts/manage_year.py` のヘッダーコメントを参照してください。

## まとめ

**推奨アプローチ**:
1. **短期**: 現在の実装を維持（個人情報を削除済み）
2. **中期**: 年度別ファイル + 設定ファイル方式を導入
3. **長期**: 個人情報を完全に分離し、公開リポジトリから除外

**重要なポイント**:
- 発表会前は個人情報を表示、発表会後は自動的に非表示
- 過去年度のデータは常に非表示
- 個人情報を含むファイルはGit管理から除外することを検討
