# 本番プレゼンテーション処理手順書

本番のPPTXファイルを15-30分で処理し、ウェブサイトに反映する手順です。

## 前提条件

- LibreOfficeがインストールされていること
  - macOS: `brew install --cask libreoffice`
  - 確認: `which libreoffice` または `which soffice`

## 手順

### 1. 本番PPTXファイルの準備（5分）

1. 本番のPPTXファイルを以下のフォルダに配置します：
   ```
   assets/presentations/final_input/
   ```

2. ファイル名に学籍番号（7桁の数字）が含まれていることを確認してください。
   - ✅ 良い例: `1522034_本番プレゼン.pptx`, `プレゼン_1522034.pptx`, `1522034.pptx`
   - ❌ 悪い例: `プレゼン.pptx`（学籍番号なし）

### 2. スクリプトの実行（5-10分）

プロジェクトルートから以下のコマンドを実行：

```bash
python3 scripts/convert/process_final_presentations.py
```

スクリプトは自動的に以下を実行します：
1. PPTXファイルを検索
2. ファイル名から学籍番号を抽出
3. PDFに変換
4. 学籍番号でリネーム（例: `1522034.pdf`）
5. `assets/presentations/rehearsal/pdf/` に配置
6. Excelファイルのプレゼンパスを更新

### 3. 結果の確認（5分）

#### 3.1 スクリプトの出力を確認

スクリプト実行後、以下の情報が表示されます：
- 処理されたファイル数
- 成功/失敗の件数
- 処理された学籍番号のリスト

#### 3.2 ファイルの確認

```bash
# PDFファイルが正しく配置されているか確認
ls -la assets/presentations/rehearsal/pdf/*.pdf

# 学籍番号でリネームされているか確認
ls assets/presentations/rehearsal/pdf/ | grep -E '^\d{7}\.pdf$'
```

#### 3.3 Excelファイルの確認

`data/students.xlsx` を開いて、以下の列を確認：
- **プレゼンパス** 列に `1522034.pdf` のような形式で記載されているか
- すべての学生のプレゼンパスが正しく設定されているか

### 4. ウェブサイトの確認（5分）

1. ウェブサイトをリロード
2. 各学生の詳細ページでプレゼンテーションPDFが正しく表示されるか確認
3. PDFビューアーが正常に動作するか確認

## トラブルシューティング

### 学籍番号が検出されない場合

**症状**: `✗ 学籍番号が見つかりません` というエラー

**対処法**:
1. ファイル名に7桁の数字（学籍番号）が含まれているか確認
2. ファイル名を変更して学籍番号を含める
   - 例: `プレゼン.pptx` → `1522034_プレゼン.pptx`

### PDF変換に失敗する場合

**症状**: `✗ PDF変換に失敗しました` というエラー

**対処法**:
1. LibreOfficeがインストールされているか確認
   ```bash
   which libreoffice
   which soffice
   ```
2. PPTXファイルが破損していないか確認
3. ファイルが他のアプリケーションで開かれていないか確認

### Excel更新に失敗する場合

**症状**: `⚠️ Excel更新をスキップしました` という警告

**対処法**:
1. `data/students.xlsx` が開かれていないか確認（閉じてから再実行）
2. Excelファイルの「プレゼンパス」列が存在するか確認
3. 手動でExcelファイルを更新：
   - 該当する学生の行を探す
   - 「プレゼンパス」列に `学籍番号.pdf` を入力（例: `1522034.pdf`）

### 既存のPDFファイルがある場合

スクリプトは自動的に既存のPDFファイルを `.backup` 拡張子でバックアップします。
- 例: `1522034.pdf` → `1522034.pdf.backup`

必要に応じてバックアップファイルを削除してください。

## 一括処理の例

複数のファイルを一度に処理する場合：

```bash
# 1. すべてのPPTXファイルをfinal_inputフォルダに配置
cp /path/to/presentations/*.pptx assets/presentations/final_input/

# 2. スクリプトを実行
python3 scripts/convert/process_final_presentations.py

# 3. 結果を確認
ls assets/presentations/rehearsal/pdf/*.pdf
```

## 注意事項

- **バックアップ**: 重要なファイルは事前にバックアップを取ってください
- **ファイル名**: 学籍番号が含まれていないファイルは処理されません
- **Excelファイル**: スクリプト実行中はExcelファイルを開かないでください
- **時間**: 大量のファイルがある場合、変換に時間がかかる場合があります（1ファイルあたり30秒〜2分程度）

## 緊急時の手動処理

スクリプトが使用できない場合の手動処理手順：

1. **PPTXをPDFに変換**
   - LibreOfficeでPPTXファイルを開く
   - 「ファイル」→「PDFとしてエクスポート」
   - ファイル名を `学籍番号.pdf` に変更（例: `1522034.pdf`）

2. **PDFファイルを配置**
   - `assets/presentations/rehearsal/pdf/` フォルダに配置

3. **Excelファイルを更新**
   - `data/students.xlsx` を開く
   - 該当する学生の「プレゼンパス」列に `学籍番号.pdf` を入力

## サポート

問題が発生した場合は、以下を確認してください：
- スクリプトのエラーメッセージ
- ファイルの配置場所
- LibreOfficeのインストール状況
- Excelファイルの構造
